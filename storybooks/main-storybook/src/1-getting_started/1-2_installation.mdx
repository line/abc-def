## Installation

ABC Def uses **Tailwind CSS v4**, which introduces a CSS-first configuration approach. Unlike v3, you don't need a `tailwind.config.js` file — all configuration happens directly in your CSS.

### 1. Install Tailwind CSS v4

Install Tailwind CSS v4 and the PostCSS plugin:

```bash
npm install -D tailwindcss@^4 @tailwindcss/postcss
# or
pnpm add -D tailwindcss@^4 @tailwindcss/postcss
# or
yarn add -D tailwindcss@^4 @tailwindcss/postcss
```

### 2. Install ABC Def Tailwind CSS Plugin

Install `@line/abc-def-tailwindcss` as a dev dependency:

```bash
npm install -D @line/abc-def-tailwindcss
# or
pnpm add -D @line/abc-def-tailwindcss
# or
yarn add -D @line/abc-def-tailwindcss
```

### 3. Configure PostCSS

Create or update your `postcss.config.js` (or `.mjs`, `.cjs`):

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

### 4. Set Up Your CSS Entry Point

In your main CSS file (e.g., `src/app/globals.css` or `src/index.css`), import Tailwind CSS and add the ABC Def plugin:

```css
@import "tailwindcss";
@plugin "@line/abc-def-tailwindcss";
```

#### Optional: Configure Dark Mode

To enable dark mode with the `.dark` class:

```css
@import "tailwindcss";
@plugin "@line/abc-def-tailwindcss";

@custom-variant dark (&:is(.dark *));
```

#### Optional: Specify Content Paths

By default, Tailwind CSS v4 automatically detects content files. If you need to customize the paths:

```css
@import "tailwindcss";
@plugin "@line/abc-def-tailwindcss";

@source "./src/**/*.{js,ts,jsx,tsx}";
@source "./components/**/*.{js,ts,jsx,tsx}";
```
