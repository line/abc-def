# Guides

Use this directory as the contributor-facing documentation entry point for the current design-system contract.

## Reading Order

1. `design-system-architecture.md`
2. `theme-and-token-contract.md`
3. `component-onboarding.md`

## Guide Map

- `design-system-architecture.md`: explains where the shared styling contract lives and how `styles`, `react`, `vue`, and the example apps fit together
- `theme-and-token-contract.md`: defines the current token layering, `@theme` alias contract, and dark-mode rules
- `component-onboarding.md`: explains how new shared components are translated into tokens, selectors, framework wrappers, and verification
